## Overview of the Updated Workflow

All scripts are written in MATLAB (MATLAB R2018a or later recommended).

The updated main script performs a nonlinear verification based on the EMTSimulator DAE solver.
It replaces the earlier linearized closed-loop simulation as the primary demonstration path.

The script in [main.m](main.m) executes the following sequence:

1. Build element and network lists from a MATPOWER case.
2. Map all non-connection buses to VSG models.
3. Reconstruct the network synchronization-reversal matrix Hnet from the power-flow operating point.
4. Compute the minimum stability requirement using the criterion in Theorem 1.
5. Simulate two scenarios:
   - a violation case (parameters below the required threshold-0.1), and
   - an adjusted case (parameters above the required threshold+0.1).
6. Compare eigenvalue-based small-signal indicators and nonlinear transient responses.

## Main Result Interpreted from the Updated Figures

The updated simulation figures generated by [main.m](main.m) show the following qualitative conclusion:

1. When the grid-code condition is violated, bus-angle trajectories diverge and bus voltages collapse or exhibit severe instability.
2. When the grid-code condition is satisfied, both angle and voltage trajectories remain bounded and stable.

This behavior is fully consistent with the theoretical stability condition derived in the paper.

## How to Run

1. Install MATPOWER and ensure it is on the MATLAB path.
2. Open MATLAB in the repository root directory.
3. Run [main.m](main.m).

The script will:

1. print scenario diagnostics in the MATLAB command window, and
2. produce a four-panel figure comparing violation versus adjusted cases in terms of bus angles and bus voltages.

## Configurable Parameters in main.m

The most frequently modified inputs are:

1. casename: MATPOWER benchmark identifier (for example, case39, case118).
2. loadscaling: multiplicative load scaling factor.
3. VSG paramters: include d and k in the scripts.
